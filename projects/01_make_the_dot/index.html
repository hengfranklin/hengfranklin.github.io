Make The Dot

Sketch-to-image apparel render system. SDXL + ControlNet-Union + custom wash LoRAs, served on H100 with TensorRT.

Role: Applied ML Engineer · Make The Dot

Dates: Sep 2025 – Jan 2026

Affiliation: Make The Dot

Category: Computer Vision & ML

Overview

Make The Dot is an AI apparel design system that turns a designer sketch, a target color, and a text prompt into a photorealistic garment render in seconds. I owned the GPU inference stack, the LoRA training pipeline, and the MLOps integration that ships new washes to production with zero downtime.

Generated denim render produced from a sketch + hex color + wash specification.

Why It Exists

Apparel teams need to test many visual directions from the same silhouette under tight timeline pressure. Manual redraw and photo sampling loops are expensive, slow, and hard to scale across categories and wash variants. This system preserves designer control from sketch input while reducing iteration time from days to seconds.

What It Does

A user uploads a flat sketch, selects an apparel category, wash or style, and target color. Seconds later, the system returns a studio-quality product render generated by Stable Diffusion XL, conditioned by ControlNet, and styled with a wash-specific LoRA adapter trained on curated reference data.

How It Works

1. Sketch In

A designer provides a flat sketch like one of these.

Input A: pants sketch used for conditional generation. Input B: polo sketch used for conditional generation.

2. ControlNet-Union Conditioning

Instead of requiring the client to precompute multiple control maps, the GPU worker derives them server-side from a single sketch input.

  • Scribble: the original sketch.
  • Canny edges: generated with controlnet_aux.CannyDetector, thresholds 50 and 100.
  • Depth map: generated with ZoeDepth from lllyasviel Annotators.

All three feed one ControlNet-Union model with per-modality scales, approximately [0.3, 0.25, 0.6] for scribble, Canny, and depth. One model and one forward pass are cheaper than stacking three separate ControlNets.

Canny edge map derived from a sketch.

3. SDXL + LoRA Generation

Stable Diffusion XL (Juggernaut-X-v10) runs inside Triton as a TensorRT-compiled pipeline. The selected wash LoRA adapters are merged into the U-Net at engine refit time, so each wash keeps a distinct visual signature without doubling VRAM use.

4. Post-Processing

  • RMBG-2 background removal for clean white studio output.
  • LAB color-space correction to nudge the generated wash toward the target hex.
  • Upload to GCS and return a presigned URL.

Same Sketch, Multiple Washes

One pants sketch and four wash LoRAs, generated by the same pipeline. Designers can iterate on one silhouette across a collection without redrawing.

Light, medium, dark, and black wash LoRAs applied to the same sketch.

Other Apparel Categories

The same pipeline generalizes beyond denim to knits, polos, pajama tops, dresses, jackets, skirts, and more.

Production Engineering

  • ControlNet-Union refactor: replaced the old multi-image client API with a single sketch input. The server derives all three control signals (scribble, Canny edges, ZoeDepth depth map) and passes them to one ControlNet-Union model with per-modality scales.
  • Multi-LoRA weighted blending: configuration-driven recipes that combine multiple wash LoRAs at inference time to create new variations (e.g. faded = light @0.9 + bleach @0.3) without retraining.
  • TensorRT engine pipeline: SDXL U-Net, VAE, and text encoders compiled to TensorRT with FP16, optimization level 5, and CUDA graphs for fast H100 inference.
  • Zero-downtime LoRA deployment: workflow for shipping new wash LoRAs to production through dedicated H100 processing, Hyperdisk ML staging, and hot volume swap into GKE inference pods.
  • Triton Python backend serving: the full diffusion pipeline wrapped as a Triton model for gRPC serving, health checks, versioning, and instance grouping.
  • FastAPI gateway: Auth0 + JWT verification, asyncpg with Postgres on RDS, Celery + Redis queueing, idempotency, rate limiting, and presigned S3/GCS result URLs.
  • Mass testing harness: a 560-task matrix across apparel, sketches, colors, and pipelines runs parallel evaluations against the live API and produces HTML A/B diff reports for promotion decisions.
  • Vertex AI custom job runner: on-demand, scheduled, and HTTP-triggered H100 evaluation batches without keeping inference clusters scaled at full capacity.
  • Data pipeline: modular vendor scrapers (Mood Fabrics, Britex), background removal with rembg, Gradio + SerpAPI reverse image search tools for curation, GPT-4 Vision captioning, and HuggingFace Diffusers LoRA training that produced 8+ wash-specific adapters.
  • Operational runbooks: procedures for GKE setup, GPU node pools, Hyperdisk attach, LoRA processing, gateway deployment, and managed SSL.

Outcomes

  • Expanded the denim wash catalog for client teams from a small base set to a broader library including light, medium, dark, black, stone, bleach, acid, purple, rinse, and vintage-aged variants. Delivered by owning the full loop: data scrape and curation, GPT-4 Vision auto-captioning, SDXL LoRA training, mass testing validation, and Hyperdisk-based production rollout.
  • Improved existing washes over multiple releases by re-curating datasets with tighter filtering, better captions, and more consistent lighting and backgrounds. Updated LoRAs were validated through side-by-side A/B runs in the 560-task harness before promotion.
  • Cut client integration complexity by deriving all ControlNet inputs server-side from a single sketch.
  • Reduced LoRA update time from multi-minute engine rebuild cycles to seconds through TensorRT refit.
  • Designed deployment with Hyperdisk ML and rolling restart so a new wash LoRA can ship with zero downtime.
  • Built an A/B harness that converted subjective pipeline comparison into a 560-image side-by-side report.
  • Evolved the data pipeline from manual scraping to automated scrape, background removal, GPT-4V captioning, and training.
